# Retail-Data-Analytics-Project-SQL-Python-Pandas-Malplotlib-Excel

## Project Overview
This project performs a comprehensive **Retail Data Analysis** to extract actionable insights from sales transactions, identify trends, and generate reports for decision-making.  

The analysis focuses on:
- Understanding sales patterns over time  
- Identifying top-selling products and categories  
- Observing customer purchase behavior  
- Automating report generation and visualizations  

---

## Project Workflow

The project was developed in the following stages:

### 1. Data Collection
- Retail transaction data was collected in CSV format (`Retail_data_transaction.csv`).  
- Relevant fields include **Transaction ID, Product Name, Category, Quantity, Price, Date**, etc.  

### 2. Data Storage
- The data was imported into an **SQLite database** (`retail_data.db`) for structured querying and analysis.  
- SQL queries (`retail_data_analysis.sql`) were used to extract required datasets for analysis.

### 3. Data Cleaning & Preprocessing
- Handled missing values, duplicates, and inconsistent formatting.  
- Converted date fields to proper datetime objects for trend analysis.  
- Prepared dataframes suitable for visualizations and reporting using Python (`pandas`).

### 4. Analysis & Visualization
- Python (`pandas`, `matplotlib`) was used to analyze data trends:  
  - Monthly sales trends  
  - Top products by sales  
  - Category-wise sales distribution  
- Generated charts to visualize insights.  

**Example: Monthly Sales Trend**
![Monthly Sales Trend](Monthly%20sales%20trend.png)

### 5. Report Generation
- Automated **Excel report** creation (`Retail_Report.xlsx`) using Python (`openpyxl`).  
- Report includes:  
  - Summary tables  
  - Sales trends  
  - Key insights  
- Ready for stakeholder presentation.

### 6. Presentation
- A **PowerPoint presentation** (`Retail_Data_Analysis_Project_presentation.pptx`) summarizes the analysis and findings.  
- Includes visuals, key insights, and recommendations.
